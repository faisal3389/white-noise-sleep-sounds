The #1 mistake with free assets is mixing sources, which creates a "frankenstein" look. This guide gives you one primary source per asset type so everything feels cohesive and professional.

Why AI-Generated Images Beat Stock Photos for This App

Uniform Art Direction

When you use the same AI model with a consistent prompt style, every image shares the same color temperature, lighting approach, and artistic feel. Stock photos from Unsplash/Pexels are taken by different photographers with different cameras and styles — they'll never look as cohesive.

Portal-Inspired Aesthetic

Your PRD calls for "moody, cinematic, slightly desaturated" backgrounds. AI lets you bake this into every prompt with phrases like "cinematic, dark moody atmosphere, desaturated colors, film grain" — and you get exactly that, every time.

No License Worries

ChatGPT and Gemini image outputs are yours to use commercially with no attribution required. No hunting through license fine print.

Step-by-Step: Getting All Sound Files

1

Go to Pixabay Sound Effects

This is your primary source. All sounds are free for commercial use, no attribution required, and the quality is consistently good. Search using the exact terms in the table below.

2

If Pixabay doesn't have what you need, use Freesound.org as backup

Freesound has the largest library. Filter by license: choose "Creative Commons 0" (CC0) only — this means no attribution required. Avoid CC-BY unless you want to add credits.

3

Download as MP3 or WAV, then convert to .m4a in Audacity

Open Audacity (free, audacityteam.org). For each sound:

  1. Open the downloaded file
  2. Trim to 30-60 seconds of the best section
  3. Make it a seamless loop: select last 2-3 seconds → Edit → Duplicate onto a new track → time-shift to overlap the beginning → use "Crossfade Tracks" effect → flatten/mix
  4. Export as: File → Export → Export as M4A (AAC)
  5. Settings: Quality = 128 kbps, Channels = Stereo, Sample rate = 44100 Hz
  6. Name the file exactly as shown in the table below
4

Test the loop

In Audacity: Edit → Preferences → Playback → check "Loop Play". Hit Play and listen for clicks or jumps at the loop point. If you hear a click, redo the crossfade.

Alternative: Quick Seamless Loop Method

If crossfading feels complex, use this trick: find sounds on Pixabay/Freesound that are already labeled as "loop" or "ambient loop." Many uploaders specifically create seamless loops. Search for terms like "rain loop" or "campfire ambient loop". These often need zero editing — just trim and export.

Audio Processing — ffmpeg Workflow

Setup: ffmpeg is installed locally

You have ffmpeg installed on your Mac via Homebrew. This is the only tool you need — one command per sound file, no GUI apps required.

Your Workflow

The Process (What We've Been Doing)

  1. Search Pixabay → find the right sound → download MP3
  2. Drop the MP3 into Raw assets/ folder in the project
  3. Claude processes it with ffmpeg → trimmed, faded, normalized, converted to .m4a
  4. Output lands in Raw assets/processed/ → ready for Xcode

How to Pick the Right Sound on Pixabay

When you search Pixabay Sound Effects and get 50+ results:

Selection Criteria

  • Duration: 30 seconds to 3 minutes — you need at least 30s of clean audio. Longer gives more material to pick the best section from.
  • Look for "loop" in the title — search "light rain loop" instead of just "light rain". Pre-made loops need less editing.
  • Listen before downloading — click play on Pixabay. You want even, consistent sound. No sudden volume changes, no voices, no music mixed in.
  • Check the waveform — a flat, even waveform = a sound that loops well. Big spikes = problems.
  • Avoid clips under 20 seconds — too short to loop cleanly.

The ffmpeg Command (What Happens Behind the Scenes)

For each sound, Claude runs a command like this:

ffmpeg -y -ss 10 -i "Raw assets/calming-rain-loop.mp3" \
  -t 45 \
  -af "afade=t=in:d=3,afade=t=out:st=42:d=3,loudnorm=I=-14:TP=-2:LRA=11" \
  -c:a aac -b:a 128k -ar 44100 -ac 2 \
  "Raw assets/processed/light_rain.m4a"

Here's what each part does:

FlagWhat it does
-yOverwrite output file if it exists
-ss 10Start at 10 seconds in (skip intros/fade-ins). Placed BEFORE -i for reliable seeking
-i "file.mp3"Input file (the raw Pixabay download)
-t 45Take 45 seconds of audio (sweet spot: long enough to not feel repetitive, small file size)
afade=t=in:d=3Fade in over 3 seconds (smooth loop start)
afade=t=out:st=42:d=3Fade out starting at second 42, over 3 seconds (smooth loop end)
loudnorm=I=-14:TP=-2:LRA=11Normalize volume to -14 LUFS — makes ALL sounds the same perceived volume
-c:a aacEncode as AAC (the .m4a codec iOS uses natively)
-b:a 128k128 kbps bitrate (good quality, small file)
-ar 4410044.1 kHz sample rate (CD quality)
-ac 2Stereo (2 channels)
Important: High Dynamic Range Sounds (Thunder, etc.)

For sounds with big volume swings (thunderstorms, fireworks), loudnorm can crush the audio to near-silence. In those cases, we skip loudnorm and use only fades. The -ss flag must go before -i (input seeking) — placing it after -i causes silent output when combined with audio filters on some files.

Adjustments for Short Source Files

Not every Pixabay clip is 3 minutes long. Here's how the command adapts:

Source DurationWhat ChangesExample
2+ minutesStandard: -ss 10 -t 45, 3s fadesforest.m4a → 45s, ~700 KB
1-2 minutesEarlier start: -ss 5 -t 45crickets.m4a → 45s, ~700 KB
30-60 secondsShorter clip, shorter fades: -ss 2 -t 27, 2s fadeswind.m4a → 27s, ~430 KB
Under 30 secondsTake almost all of it: -ss 2 -t 26, 2s fadesheavy_rain.m4a → 26s, ~414 KB

You can also run these commands directly in your Mac Terminal. Just cd into the project and run:

File Size Reference

Target sizes per file:

DurationExpected .m4a Size (128kbps)Status
30 seconds~250-350 KBGood
45 seconds~400-500 KBIdeal
60 seconds~500-700 KBGood
90+ seconds~900+ KBToo large

Quick math: 128 kbps AAC ≈ 16 KB per second. A 45-second file = ~720 KB theoretical, ~400-500 KB actual (AAC compresses efficiently).

If a file is too large:

  • Trim shorter (45 seconds is the sweet spot)
  • Drop to 96 kbps: change -b:a 128k to -b:a 96k (still sounds good for ambient)
  • Go mono for machine sounds (fan, AC, dryer): add -ac 1 to the ffmpeg command

Total budget: 31 files × ~450 KB = ~14 MB. Well under the 200 MB App Store limit.

Troubleshooting

"command not found: ffmpeg"

ffmpeg isn't installed. Run: brew install ffmpeg. If brew isn't found either, install Homebrew first: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

"Click" or "pop" at the loop point

The fade duration is too short. Increase from 3 to 5 seconds in your ffmpeg command: afade=t=in:d=5,afade=t=out:st=40:d=5 (and adjust -t accordingly).

File is too large (over 700 KB)

Trim shorter (45 seconds is the sweet spot), or drop bitrate to 96k: -b:a 96k

Output file is silent or tiny (under 30 KB)

This happens with high dynamic range audio (thunderstorms, etc.) when loudnorm is used. Remove loudnorm from the filter chain and rely on just afade. Also make sure -ss is placed before -i (input seeking), not after.

Thumbnails (35) & Scene Cards (8)

Thumbnails — Crop from Backgrounds

These are free — they come from your backgrounds

Don't generate thumbnails separately. Crop the center of each background image to a 120×120px square. This keeps the visual language 100% consistent.

1

Batch crop with a Terminal command (after you have all backgrounds):

# In Terminal, cd to the folder with your background JPEGs:
mkdir thumbnails
for f in bg_*.jpg; do
  name=$(basename "$f" .jpg)
  # Crop center 120x120 from the image
  sips -c 1290 1290 "$f" --out "temp_$f"
  sips -z 120 120 "temp_$f" --out "thumbnails/thumb_${name#bg_}.png"
  rm "temp_$f"
done

This crops each background to a center square, then resizes to 120×120 PNG.

2

Or use Preview (manual): Open image → Tools → Adjust Size → make it square → resize to 120×120 → Export as PNG.

Background Images (Asset Catalog)

1

Open Assets.xcassets in Xcode

Click on Assets.xcassets in the Xcode navigator.

2

Create a "Backgrounds" folder group:

  1. Right-click in the asset catalog sidebar → New Folder
  2. Name it Backgrounds
3

Drag all background JPEGs into the Backgrounds folder:

  1. Select all your bg_*.jpg files in Finder
  2. Drag them into the Backgrounds folder in the asset catalog
  3. Xcode automatically creates an .imageset for each with the file name (minus extension) as the asset name
  4. Each image lands in the "1x" slot — this is fine since we're using the full resolution
4

Verify the names match SoundLibrary.swift:

The asset catalog name must exactly match the backgroundImage string in your Sound model. For example, if you drag in bg_light_rain.jpg, the asset catalog name will be bg_light_rain — which matches Sound(... backgroundImage: "bg_light_rain" ...).

Thumbnails (Asset Catalog)

Same process as backgrounds — create a Thumbnails folder in the asset catalog and drag in all thumb_*.png files. If your code uses the background image name for thumbnails too (just scaled down in SwiftUI), you may not need separate thumbnail assets at all.

App Icon

1

Open Assets.xcassets → click AppIcon

Drag your 1024×1024 PNG into the "All Sizes (1024pt)" slot. Xcode auto-generates every size from this single image.
